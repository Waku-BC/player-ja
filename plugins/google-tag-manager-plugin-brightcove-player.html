    Brightcove ブプレーヤー用の Google タグマネージャプラグイン

    このトピックでは、Brightcove Player 用の Google タグマネージャ（GTM）プラグインの使用方法について説明します。

    概要

    Googleタグマネージャー（GTM）プラグインを使用すると、GTM機能をBrightcovePlayerと統合できます。

    タグマネージャを使用すると、コードを使用せずにタグを追加および公開できます。タグマネージャは、クライアント側のソリューションであるため、さまざまなマーケティング技術にわたってデータを収集できます。これにより、マーケティング担当者は、多様な収集ポイントにわたってユーザーとプレーヤーのデータを収集できるようになります。

    すでにGoogleAnalytics、Adwords、DoubleClickを使用している場合は、これらのツールとうまく統合できるGoogleタグマネージャーも使用することをお勧めします。GTMは、多数の変数およびトリガーオプションを提供します。

    Googleタグマネージャープラグインは あなたが追跡する Brightcove ブ プレーヤー イベント を使用して  Google Analytics  ダッシュボード。 ザ・ 次のプレイヤー イベント 追跡できます： 

    • Player load
    • 動画インプレッション
    • 動画表示回数
    • ビデオ エンゲージメント
    • 再生リクエスト
    • 動画の完了
    • 高度なプレーヤーイベント といった 広告開始、 広告終了
    • 混乱させるエラー 再生体験
    Googleタグマネージャープラグインは、BrightcovePlayerバージョン6.36.5以降でサポートされています。

    タグ管理の用語

    Googleタグマネージャー（GTM）を使用するときに遭遇する一般的な用語のいくつかを次に示します。

    用語 説明
    コンテナ タグ管理システムアカウントは、独自のタグと変数を持つ複数のコンテナ (プロファイル) を持つことができます。各コンテナは複数の環境を持つことができ、通常は開発、ステージング、本番用にセットアップされます。
    データ層 データレイヤーとは、Webページのコードに存在するデータのコレクションを指します。この JavaScript オブジェクトは、Web サイトからタグマネージャコンテナに情報を渡すために使用されます。次に、この情報を使用して変数を設定し、タグ構成でトリガーをアクティブ化できます。
    イベント イベントは、レポートツールにデータを送信するために使用されるデータポイントです。これらは、ビデオの読み込みと再生に関連しています。
    タグ タグは、ユーザーインタラクションデータを収集するために Web ページに配置されるコードスニペットです。タグ管理システムを使用すると、サイトにタグを追加するプロセスが簡素化され、タグ ID とタグイベントタイプのみを指定する必要があります。
    タグマーケットプレイス タグマーケットプレイスのテンプレートは、タグを展開するためのコードスニペットを提供します。これにより、必要なコード知識の量が減りますが、実装を成功させるには JavaScript と HTML に精通している必要があります。
    トラッカー トラッカーは、データを収集、保存、Google タグマネージャに送信するオブジェクトです。
    引き金 トリガーは、タグコードをページに追加するタイミングを示します。これらはユーザー定義の条件です。

    入門編

    Googleタグマネージャを設定するには、次の手順に従います。

    ステップ

    1. Googleのドキュメントを確認してタグマネージャーを設定してインストールする。

    2. にタグマネージャー、アカウントを作成するか、既存のアカウントを使用します。新しいアカウントを作成するには、アカウントを作成するボタン。

      デフォルトでは、新しいアカウント用に新しいコンテナが作成されます。各アカウント内に追加のコンテナーを作成できます。

    3. アカウント名やコンテナ名など、新しいアカウントページの値を入力します。ターゲットプラットフォームを選択します。

    4. [ 作成 ] ボタンを選択します。

    追跡するイベントの確認

    以下の表を確認して、追跡するイベントを決定してください。

    タイプ イベント 説明
    プレーヤーのロード bcvideo_player_load プレーヤーが読み込まれ、再生の準備ができたときに送信されます
    動画インプレッション bcvideo_video_impression 動画がプレーヤーに読み込まれ、再生の準備ができたときに送信されます
    ビデオビュー bcvideo_video_view 自動再生またはユーザー操作により動画の再生が開始されたときに送信されます
    動画エンゲージメント bcvideo_video_engagement ユーザーが動画のタイムラインの秒の範囲を視聴しました。このイベントは、動画エンゲージメントを追跡するためのハートビートであり、ユーザーの操作と動画の長さによっては、再生中に何回も送信されます。
    再生リクエスト bcvideo_play_request ユーザーが再生ボタンを明示的にクリックして再生を開始したとき、またはプラットフォームが自動再生シナリオで再生をトリガーしたときに自動的に送信されます
    広告開始 bcvideo_ad_start 再生プラットフォームから広告代理店にコントロールが渡されたときに送信されます
    広告終了 bcvideo_ad_end 制御が広告代理店から再生プラットフォームに戻されたときに送信されます
    エラー bcvideo_error 再生エクスペリエンスを中断させる致命的なエラーが発生したときに送信されます
    ビデオコンプリート bcvideo_video_complete ビデオの再生が終了したときに送信されます

    変数、タグ、トリガーの作成

    追跡するすべてのBrightcoveイベントは、イベントタグ Googleタグマネージャーで。GTMのイベントIDはプレーヤー内で使用され、BrightcoveイベントデータをGTMイベントに渡すことができます。

    これらのイベントをGoogleアナリティクス（GA）に送信するには、タグごとにGTMトリガーを作成する必要があります。イベントに関連付けられたカスタムパラメータもGTMで作成されます変数タブ。

    GTMへの変数の追加

    パラメータは追加のデータを格納し、イベントとともに渡されます。記録されるすべてのBrightcoveパラメータは、Googleタグマネージャーで変数として作成する必要があります。指定されたパラメータはプレーヤー内で使用され、BrightcoveパラメータをGoogleタグマネージャー変数に渡すことができます。

    Googleタグマネージャで変数を作成するには、次の手順に従います

    1. Google タグマネージャーアカウントにログインする

    2. 選択する変数左側のナビゲーションで。
    3. にとってユーザー定義変数、 を選択新着ボタン。

    4. を選択変数タイプなのでデータレイヤー変数。

    5. 詳細を入力します。

      • データレイヤー変数名 -変数の名前
      • データ層 バージョン -バージョン2

    6. 作成するパラメータ：

      • アカウント -Brightcoveアカウント名
      • プレーヤー -プレイヤーID
      • プレーヤの名前 - プレーヤの名前
      • プラットフォームバージョン -プラットフォームのバージョン番号
      • セッション -セッション識別子
      • ビデオ -ビデオID
      • ビデオ名 -ビデオの名前
      • 範囲 -StartSecond..EndSecondの形式で表示される秒の範囲
      • 視聴したビデオ秒数 -視聴された秒数
      • ビデオの長さ -秒単位のビデオの長さ
      • 視聴されたビデオの割合 -視聴されたビデオの割合
      • エラーコード -イベントに関連するプラットフォーム固有のエラーコード
      • ビデオマイルストーン -ビデオで完成したマイルストーン（25％、50％、75％、100％）
      • ビデオ再生ヘッド -イベントがトリガーされるビデオ内の時間
      • 国 -ISO-3166（アルファ2）国コード、 例えば：米国
      • 国の名前 -人間が読める国の名前 例えば：米国
      • 領域 -ISO-3166（アルファ2）リージョンコード
      • 地域名 -人間が読める地域の名前
      • 市 - 市の名前
      • ブラウザ -使用されているブラウザのフルネームとバージョン
      • ブラウザの種類 -使用されているブラウザの一般名（Chrome、Edge、Firefoxなど）
      • 先 -イベントを発生させたURI
      • ソース -エンドユーザーを宛先に送信したURI
      • デバイスタイプ -ユーザーエージェントが信頼できない場合に、イベントを発生させたデバイスのOSを指定するようにオーバーライドします
      • デバイスOS -デバイスで使用されているOSの名前
      • デバイスOSバージョン -デバイスで使用されているOSのバージョン

    7. ユーザー定義変数は、GTMでは次のようになります。

    GTMへのタグの追加

    追跡するイベントを決定したら、Googleタグマネージャーでタグを作成する必要があります。

    Googleタグマネージャでタグを作成するには、次の手順に従います

    1. Google タグマネージャーアカウントにログインする

    2. 選択するタグ左側のナビゲーションで、を選択します新着ボタン。

    3. を選択タグタイプなのでグーグルアナリティクス：ユニバーサル分析。

    4. タグの詳細を入力します。

      • トラックタイプ - に設定イベント
      • カテゴリー -イベントカテゴリ変数
      • アクション -イベントアクション変数
      • ラベル -イベントラベル変数
      • 値 -なし
      • 非インタラクションヒット - に設定本当
      • GoogleAnalyticsの設定 - 初期設定
      • トラッキングID -GAダッシュボードのトラッキングID。トラッキングID変数を選択します。

    カスタム寸法の作成

    カスタムディメンションは、Googleアナリティクスで自分で作成する指標です。詳細については、Googleのカスタムディメンションと指標資料。

    Google Scriptsを使用すると、GoogleAnalyticsでカスタムディメンションを追加するプロセスを自動化できます。

    ステップ

    Google App Scriptを使用してカスタムディメンションを作成するには、次の手順に従います。

    1. Googleにログインします。

    2. に移動しますGSuite開発者ハブ AppsScriptの新しい空白のプロジェクトを選択します。

    3. 選択する無題のプロジェクトプロジェクトに名前を付けます。

    4. 次のカスタムアプリスクリプトコードをエディターにコピーし、次の値を更新します。

      • accountId - Your Google Analytics account ID
      • webPropertyId - Your Google account tracking ID
      • dimensions - The custom dimensions to be added.現在のリストには、すべてのカスタムディメンションが含まれています。寸法の前に接頭辞が付いていることに注意してくださいbcvideo_。これは、これらの変数との名前の競合がないことを確認するためです。 
      //Set values for GA Account ID and web property ID to which you have to set custom dimensions.
var accountId='your account id';
var webPropertyId='your tracking code';
//Add required custom dimensions to be created to this list
var dimensions=['bcvideo_video_seconds_viewed','bcvideo_video_playhead','bcvideo_account','bcvideo_player','bcvideo_player_name','bcvideo_video','bcvideo_video_name','bcvideo_session','bcvideo_platform_version','bcvideo_range','bcvideo_video_duration',
'bcvideo_video_percent_viewed','bcvideo_error_code','bcvideo_milestone','bcvideo_destination'];


//Function to create custom dimensions in GA
function createGACustomDimensions() {

  //This loop iterates through the custom dimensions in order to set the custom dimension scope, active values into our GA property
  for(var index=0;index<dimensions.length;index++){
  var customDimensions ={
    'name': dimensions[index],
    'scope':'HIT',
    'active':true,
  }

  //This create a request to the new custom dimension
  Analytics.Management.CustomDimensions.insert(customDimensions,accountId,webPropertyId);

  // Pause for 1 second between adding each custom dimension, this ensure that the GA property has time
  // to process the request fully before moving to the next custom dimension.
  Utilities.sleep(1000)
  }
}

    イベントとパラメータ

    次のイベントとパラメータをGoogleタグマネージャーに送信できます。

    イベント

    これらのイベントは、あなたのプレーヤーから来ます。

    • player_load
    • video_impression
    • play_request
    • video_view
    • video_engagement
      • video_engagement - 25% Milestone
      • video_engagement - 50% Milestone
      • video_engagement - 75% Milestone
    • ad_start
    • ad_end
    • error
    • video_complete - 100% Milestone

    ビデオ視聴率（video_engagement）は次のように計算できます（GoogleおよびAdobeアナリティクスのvideo_engagementイベントと同様）。 

    Video watched percentage = duration the user actually watched / total video duration

    例えば、20秒のビデオでは、以下の両方のケースで25％のマイルストーンを達成することができます

    • ユーザーは0～5秒から監視します
    • ユーザーは0-1秒から見て11秒にスキップして15秒に観る

    上記のいずれの場合も、ユーザーは 5 秒を視聴したため、マイルストーンは 25% として報告されます。
